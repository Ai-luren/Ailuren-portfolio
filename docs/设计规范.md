# 当前设计规范

## 视觉方向

页面整体是“深空创作舱”：用全屏视频建立叙事环境，用冷蓝信号色标记章节和交互，用半透明深色面板承载信息。

## 视觉规则

- 背景：`assets/videos/hero.mp4` 负责空间感，内容层使用深蓝/黑色遮罩保证可读性。
- 颜色：正文使用冷白和浅灰，辅助信息使用灰蓝，重点交互使用亮蓝 `#2997ff`。
- 字体：中文使用 `Noto Sans SC`；编号、英文标签和短状态使用 `DM Mono`。
- 面板：使用低饱和深色玻璃、细描边和轻微内阴影；不要重复叠加多个边框。
- 圆角：统一三档——控件 `8px`、卡片 `12px`、胶囊 `999px`；圆形只用于品牌图标和社交入口。
- 标题：每个章节保留一个中文主标题和一个英文编号标签，避免增加无法解释的装饰标题栏。

## 布局规则

- 内容版心由 `--content-max` 和 `--content-gutter` 控制，模块之间保持统一节奏。
- 章节结构优先使用“章节标题 → 主要内容”，不额外添加泛化的横向信息栏。
- 工作经历使用“左侧阶段列表 → 中央进度线 → 右侧详情”的滚动叙事结构，滚动时固定内容区并切换阶段。
- 页脚使用居中布局：上方品牌名居中，中间图标导航 dock 居中，下方版权声明；桌面端额外显示文字导航。
- 移动端隐藏文字导航，保留品牌名和图标导航居中排列。

## 交互规则

- 交互优先使用位移、透明度和边框变化，不使用强烈扫光或持续闪烁。
- 滚动固定模块使用 GSAP ScrollTrigger，进度动画必须使用 `scrub`，不要添加调试 markers 到生产页面。
- 作品卡片可使用轻微鼠标倾斜和 `translateZ` 分层，但文字默认态与悬停态必须保持稳定可读。
- 所有可操作元素需要有键盘焦点状态。
- `prefers-reduced-motion: reduce` 下关闭非必要位移和循环动画。
- 入场动画统一使用 `ANIM` token 对象管理参数（intro / hero / stage / footer 四类），避免散落的魔法数字。

## 文案规则

- 面向面试官和猎头，优先具体、直接、可验证的表达。
- 少用“赋能、闭环、影响力证明、能力验证”等模板化词语。
- 中英文并置只用于章节标签、工具类别和必要的导航信息。
