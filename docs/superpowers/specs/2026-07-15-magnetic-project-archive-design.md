# 精选创作档案：磁吸轮播设计

## 目标

把当前的 3D 档案轮替换为横向磁吸封面带：鼠标靠近时封面按距离扩张，点击后聚焦当前作品。页面仍以视频为主视觉，轮播只承担项目入口，不新增外部运行时依赖。

## 内容与素材

- 首批保留 6 个档案位。
- 已有两张真实封面：`AI作品集封面/无声的回响封面.webp`、`AI作品集封面/人河流城市.webp`。
- 其余四张使用统一的本地“待补充档案”占位封面；替换时只需替换对应图片路径，不改交互代码。
- 每张档案保留编号、标题、摘要和渠道字段；未补全信息使用“待补充”，不伪造项目内容。

## 桌面交互

- 默认：六张竖向窄封面居中排列。
- 指针靠近：当前封面向 1.9 倍宽度、略增高度；相邻封面按距离递减扩张。
- 点击：当前封面展开为横向主封面，其他封面弱化但不使用重模糊；再次点击、点击关闭控件或按 Escape 恢复。
- 键盘：Tab 可聚焦封面；Enter/Space 展开；Escape 收起；左右方向键切换当前档案。

## 移动端与降级

- 移动端不使用依赖鼠标距离的磁吸，改为横向滚动吸附；点按卡片展开。
- `prefers-reduced-motion` 下取消连续磁吸跟随，只保留即时焦点与展开状态。

## 视觉约束

- 使用现有深空蓝黑底、细边线与单色蓝交互强调；不使用参考组件的外链图片或默认色。
- 卡片圆角克制，正文与标题放在封面下方的渐隐阅读区，保证视频背景下可读。
- 以现有 Vite + React 运行时实现为独立的 `MagneticProjectArchive` 组件，并在 `#projects` 区块挂载；不额外引入 Tailwind 或 Framer Motion。
- 现有 `index.html` 继续承担视频背景与其他章节，原有档案轮盘的 HTML、CSS、JS 在组件接管后移除，避免两套交互并存。

## 验收

- 桌面端鼠标靠近时放大流畅、没有布局抖动。
- 点击、键盘、移动端横滑和 reduced-motion 均能正常工作。
- 所有封面为本地资源，静态服务器返回 `index.html` 200，`npm run build` 通过。
