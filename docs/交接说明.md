# 项目交接说明

## 运行方式

```bash
npm install
npm run dev
npm run build
```

开发服务默认使用 `http://127.0.0.1:5173/`。生产构建输出到 `dist/`，Vite 构建时使用相对路径以兼容 GitHub Pages 根地址和子路径。

当前线上地址：<https://ai-luren.github.io/>。

## 运行时结构

`index.html` 是页面主体，包含章节结构、全局 CSS、背景视频、GSAP/ScrollTrigger 脚本和静态文案。`src/main.jsx` 等待 `window.__gsapReady` 完成后启动 React，`src/App.jsx` 通过 portal 将交互模块挂载到页面预留节点：

- `#projects-react-root` → `MagneticProjectArchive`：精选创作档案和鼠标 3D 分层。
- `#recognition-react-root` → `ImpactEvidence`：获奖截图、平台 Logo 和四个平台内容发布卡片。
- `#footer-dock-react-root` → `FloatingDock`：页脚社交入口和复制操作。
- `#experiments-react-root` → `ExperimentFlip`：AI 实验室交互卡片。
- `#hero-actions-react-root`、`#hero-title-react-root` → Hero 操作和标题动效。
- `VideoPostProcess`、`ExperienceGlareLayers`：背景视频后处理和工作经历视觉层。
- `#profile .experience-scroll`：工作经历滚动固定模块，GSAP ScrollTrigger 根据滚动进度切换五个阶段。

其余章节目前由 `index.html` 直接渲染。

## 关键资源

- `hero.mp4`：全屏背景视频。
- `assets/fonts/`：页面使用的本地字体。
- `assets/images/`：社交图标和头像资源。
- `创作者LOGO/`、`获奖截图/`、`社交媒体主页/`：影响力模块的内容资源。
- `AI作品集封面/`：精选创作档案封面。

## 维护注意

- 修改文案时优先查找 React 数据数组或 `index.html` 对应节点，不要在构建产物 `dist/` 中直接改。
- 修改 `ImpactEvidence` 时同步检查桌面端和移动端 CSS，尤其是 Logo 尺寸、卡片底部数据和内容换行。
- 修改工作经历时同步更新 `.experience-step` 和 `.experience-detail` 的同序数据，并保留 `data-experience-index` / `data-experience-panel` 的对应关系。
- 修改工作经历滚动范围时检查 `experienceRoot` 的最小高度、ScrollTrigger 的 `end` 和移动端布局，避免固定区域过短或页面滚动过长。
- 修改作品档案 3D 效果时保留键盘焦点和 reduced-motion 分支。
- 修改 GSAP 加载或 `CometCard` 时保留 `window.__gsapReady` 等待逻辑，避免 React 初始化早于 GSAP 导致鼠标 3D 监听未绑定。
- 邮箱、电话和社交入口位于 `index.html` 页脚；邮箱图标当前执行复制邮箱操作，电话和微信图标复制对应号码。
- 不再保留旧的设计探索页、临时页脚对比页或一次性实施计划；新的长期规则只写入本目录的 `AGENTS.md` 和 `docs/`。
