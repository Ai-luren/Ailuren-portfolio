# 深空创作舱作品集｜当前设计规范

> 状态：当前执行规范
>
> 版本日期：2026-07-14
>
> 代码真值：`index.html` 负责现有滚动叙事基线；React/Vite 入口为 `src/main.jsx` 与 `src/App.jsx`。

## 1. 设计定位

这是一个以视频滚动叙事为主的个人作品集，不是后台控制台，也不是把七张科幻海报拼成一页。

页面需要让访客在首屏快速理解：

1. 宁磊从动态设计出发，负责 AI 图像与视频创意效果设计。
2. 作品集本身证明了网页编码、动效组织和滚动叙事能力。
3. 工作经历、创作档案、影响力和实验方法都有清晰入口。

## 2. 当前视觉原则

### 2.1 视频是主角

- `hero.mp4` 作为全页持续的空间背景和滚动叙事主资产。
- 文字只落在安全区域，不用大面积纯黑遮罩盖住视频。
- 背景层只负责压低高光和建立文字可读性，不制造新的视觉主体。
- 新增组件默认使用透明容器：不得自带纯黑背景、黑色蒙版或深色渐变底，避免与各章节的视频场景断层；只有明确指定为独立收尾面、弹窗或阅读承载区时才可例外。

### 2.2 每个章节只有一个主焦点

- Hero：职业标题与视频镜头。
- 设计宣言：一句判断。
- 航行记录：职业轨迹。
- 创作档案：当前作品。
- 奖项影响力：创作轨迹关系。
- AI 实验舱：方法与证据。
- 通信终端：联系方式。

装饰不能与主焦点争夺注意力。细线、节点、轨道和琥珀色信号只做结构提示。

### 2.3 薄结构，不堆 UI

- 优先使用留白、版心、细边线和背景明暗组织画面。
- 不新增大面积玻璃卡片、密集 HUD、假仪表盘和无意义状态文案。
- 不用装饰去填补真实作品、截图或数据的空缺；素材缺失时明确标记占位。

## 3. 字体与文字层级

### 3.1 字体分工

| 用途 | 当前字体 | 规则 |
| --- | --- | --- |
| 中文标题、正文、按钮 | `Noto Sans SC` | 全站统一，优先保证字形、字宽和粗细稳定 |
| 英文、年份、编号、状态标签 | `DM Mono` | 只用于短元数据，不用于中文长句 |
| 系统回退 | `PingFang SC` / `Microsoft YaHei` / `sans-serif` | 外部字体加载失败时保持可读 |

页面不再调用 Glow Sans 作为展示字体。项目中的 Glow Sans 文件暂时保留，未获确认前不删除。

### 3.2 权重规则

- Hero 主标题：`700`，两行使用同一字号、同一颜色和同一字重。
- 章节标题：`700`，只保留一个主层级。
- 宣言正文：`600`，作为章节唯一视觉焦点。
- 正文与说明：`400–500`，不使用过细字重压在复杂视频上。
- 技术标签：`500` 等宽字体，使用小字号和较宽字距。

### 3.3 当前版式要求

- 禁止通过不同字体、不同颜色和不同字号同时制造多个标题焦点。
- Hero 中 `AI 图像与视频` 与 `创意效果设计` 必须保持同一视觉等级。
- 宣言正文整句使用同一颜色和字重，不再局部变成琥珀色。
- 正文行长控制在约 30–36 个中文字符以内，避免压成一整面文字。
- 文字阴影只用于保证视频上的边缘识别，不承担装饰功能。

## 4. 色彩规则

| 角色 | 当前用途 |
| --- | --- |
| 深空黑蓝 | 页面底色、视频暗部 |
| 蓝灰 / 雾白 | 正文、标题和结构信息 |
| 低对比灰 | 次要说明、非当前节点 |
| 琥珀色 | 当前节点、真正可点击状态、短分隔线和焦点反馈 |

琥珀色是唯一信号色，不用于所有标签、所有星点或整段文字。页面的亮度主要来自视频高光和主标题。

## 5. 版心与章节节奏

- 桌面端最大内容宽度约 `1220px`，左右保持稳定边距。
- 使用 12 列结构组织章节标题与说明，不让每个模块都居中堆叠。
- 章节之间用上下留白建立呼吸，不用厚重分隔面板。
- 页面节奏为：开场 → 停顿 → 背书 → 作品高潮 → 影响力 → 方法 → 收束。
- Hero 与设计宣言留白最大；工作经历和实验舱承担信息密度；联系方式保持安静。

## 6. 模块规则

### Hero

- 左侧标题，右侧保留视频主体。
- 不使用蓝色标题与白色标题的强对比组合。
- 副标题只说明职业背景、负责范围和网页叙事能力。

### 设计宣言

- 只保留标题和一句主张。
- 主张使用稳定的中文无衬线字体，居中或在明确版心内对齐。
- 分隔线可以使用一条短琥珀线，不能叠加准星、扫描线和多层装饰。

### 航行记录

- 以五节点轨道表达职业轨迹。
- 小米、快手为重点节点；早期经历降低视觉权重。
- 节点、年份、公司和岗位必须有清晰的垂直间距。

### 创作档案

- 当前保留中心主展和外围预览的结构。
- 占位内容必须明确写“占位”或“待补充”，不伪造作品资料。
- 真实封面、年份、奖项和外链补齐后，优先让作品本身成为焦点。

### 奖项与影响力

- 星图表达关系，不承担完整奖项清单。
- 节点尺寸固定，轨道低频运动，文字默认隐藏或保持低干扰。
- 外部行业认可、内部荣誉、项目成果和内容影响力保持分组。

### AI 实验舱

- 作为方法证据区，不模拟后台产品。
- 不把没有真实来源的百分比当成测评数据。
- 真实工作流截图、评估维度或网页实现证据优先于装饰性仪表。

### 通信终端

- 联系方式比“通信终端”等系统化文案重要。
- 邮箱、电话、微信和外链必须清楚、可操作。
- 结尾不再添加大型结束口号或复杂状态动画。

## 7. 动效规则

- 页面只有一条主线：滚动推进视频和章节叙事。
- 章节入场使用 `transform` 与透明度，单次约 `260–700ms`。
- 文字以轻微上移或淡入为主，不使用大幅横向飞入。
- 轨道、星点和扫描线只做低频环境动效，离开视口时暂停。
- 不使用滚轮劫持、强制滚动、持续闪烁、鼠标跟随光球或高密度粒子。
- `prefers-reduced-motion` 下，页面仍需完整显示内容并关闭非必要动效。

## 8. 响应式与可读性

- 移动端取消高方差固定布局，优先恢复正常纵向阅读。
- Hero 标题允许自然换行，不通过压缩字宽解决溢出。
- 时间线在窄屏降为单列或垂直节奏。
- 档案、星图和实验舱在移动端保留信息，不强行保留桌面装饰结构。
- 关键按钮保留真实按钮语义、键盘焦点和可见反馈。
- 背景视频失败时，文字和交互仍必须可读可用。

## 9. 设计验收清单

- [ ] 首屏 3 秒内能读懂职业定位。
- [ ] 中文标题没有字宽压缩、字体跳变或局部粗细异常。
- [ ] Hero 只有一个标题层级，主副标题颜色关系稳定。
- [ ] 宣言文字整句统一，不被背景高光吞没。
- [ ] 琥珀色只出现在状态、焦点和真实交互上。
- [ ] 视频仍然是页面主视觉，没有被黑色面板盖住。
- [ ] 各章节有不同节奏，不是七张相同的科幻卡片。
- [ ] 桌面端和移动端没有横向溢出。
- [ ] 减少动效模式、键盘操作和视频失败回退可用。

## 10. 素材与维护边界

- 页面入口和实现以 `index.html` 为准。
- `hero.mp4`、本地 GSAP 和现有联系方式不因视觉调整被替换或删除。
- 新增作品封面、截图或视频前，先确认真实来源、命名和页面引用关系。
- 旧方案只用于追溯，不得作为新的排版、字体或交互执行依据。
