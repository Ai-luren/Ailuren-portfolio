# 项目规范

## 项目目标

这是一个可直接部署到 GitHub Pages 的单页滚动叙事作品集。优先保证首屏加载、移动端可用性、无障碍基础与外部资源失败时的可读性。

## 目录与命名

- `index.html`：页面的静态内容与既有滚动叙事视觉基线；由 Vite 作为入口处理。
- `src/main.jsx`、`src/App.jsx`：React 运行时与页面级状态入口。
- `src/components/`：可复用的 React 组件；3D 工牌在 `src/components/Lanyard/`。
- `src/assets/`：由 React 组件直接引用的贴图和占位资源。
- `public/`：不经打包处理、按原文件名发布的静态资源；目前存放原生 GSAP 运行时。
- `hero.mp4`：首屏背景视频；上线版本使用压缩后的 H.264 MP4。
- 作品配图素材：以页面实际引用的本地资源为准；当前根目录没有 `IP/` 目录，不要假设该目录存在。
- 文档使用中文文件名；网页资源使用小写英文和连字符。

## 验证

- 修改后运行 `npm run build`，并用 `npm run preview -- --host 127.0.0.1` 确认首页返回 200。
- 检查页面引用的本地资源均存在，关键按钮、工牌 Esc 关闭与键盘焦点语义完整。
- GitHub Pages 从仓库根目录发布，不使用 Git LFS 承载页面视频。

## 清理规则

- 不提交 `.DS_Store`、临时导出文件或未被页面引用的媒体文件。
- 删除文件、替换现有媒体或变更部署配置前必须先征得用户确认。

## 深入文档

- `设计规范.md`：当前版本的设计、美学、字体、排版、动效和验收规则。
- `宁磊个人简历- AI 创意效果.md`：经历、奖项与联系方式的事实来源；页面文案改动时先核对。
